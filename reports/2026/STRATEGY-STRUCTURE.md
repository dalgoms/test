### Platform: Platform Scope & AI Role `[상반기 프로젝트]`

| 섹션 | 내용 |
|------|------|
| 플랫폼 정의 | "Clipdesk는 AI 영상 편집 툴이 아닙니다" |

| AI 역할 정의 | 내부 생산성 엔진으로서의 AI |
| 경계 선언 | Clipdesk가 아닌 것 vs 지향하는 것 |
| 무료 툴 정책 | Free AI Tool Policy & Customer Acquisition |

#### Three Layer Model

```
┌─────────────────────────────────────────────┐
│  Layer 1: Tool Layer                        │
│  자동 컷, 자동 자막, 템플릿, 셀프 편집        │
│  → 개인 사용자 중심 (Clipdesk 주력 아님)      │
├─────────────────────────────────────────────┤
│  Layer 2: Workflow Layer                    │
│  영상 업로드, 자막/번역 요청, 피드백, 상태관리 │
│  → Client-facing 운영 UX                    │
├─────────────────────────────────────────────┤
│  Layer 3: Outcome Layer (Core)              │
│  목적 정의, 편집 설계, 품질 검수, 납기, 운영  │
│  → Clipdesk 핵심 가치                       │
└─────────────────────────────────────────────┘
```

#### 핵심 선언

> "Clipdesk는 AI 툴이 아니라, AI로 가속되는 결과 책임형 플랫폼입니다."

> "무료 AI 툴은 사용자를 모으는 전략이고, Clipdesk는 결과를 책임지는 플랫폼입니다. 이 둘은 같은 제품 안에 존재하지 않습니다."

## 핵심 메시지

### 브랜드 통합
- Textar / Textar Global → Clipdesk 내부 모듈로 재편
- 기능 단위 서비스 → 결과 책임형 플랫폼

### 경쟁 차별화
- AI 툴 (CapCut, Vrew) ≠ Clipdesk
- Tool 중심 ≠ Outcome 중심

### 플랫폼 정체성
- AI는 고객 도구가 아닌 내부 생산성 엔진
- 셀프 편집 툴 ❌ → 결과 책임 플랫폼 ⭕

### Growth 구조
- timbel.net = 회사 신뢰 레이어
- clipdesk.net = 유입·전환 레이어 (실제 비즈니스 엔진)
