3. 기술 구조 (개발자용)

폴더 구조

timbel-homepage/ ├── index.html # 메인 페이지 (SEO + Schema.org) ├── contact.html # 문의하기 (Formspree 연동) ├── sitemap.xml # 사이트맵 (SEO) ├── robots.txt # 크롤링 규칙 (SEO) ├── js/ │ └── common.js # 공통 헤더/푸터 컴포넌트 (핵심) ├── img/ │ └── timbellogo.png # 로고 이미지 ├── company/ # 회사소개 (5개) ├── products/ # 제품 (9개) ├── solutions/ # AICC 솔루션 (5개) ├── stenography/ # AI 기록·자막 (4개) └── resources/ # 자료 센터 (2개)

공통 컴포넌트 핵심 함수 (common.js)

함수명 역할
getBasePath() 현재 페이지 위치에 따른 상대 경로 자동 계산
getHeaderHTML(basePath) 헤더 + 메가메뉴 + 모바일 메뉴 HTML 생성
getFooterHTML(basePath) 푸터 HTML 생성
getCommonStyles() 헤더/푸터 공통 CSS 생성
initCommonEvents() 스크롤 효과, 모바일 메뉴 토글 이벤트 바인딩
renderCommon() DOMContentLoaded 시 전체 렌더링 실행

사용 방법

각 HTML 페이지에 플레이스홀더와 스크립트만 추가하면 헤더/푸터가 자동 렌더링됩니다.
<body> <div id="header-placeholder"></div> <main> <!-- 페이지 고유 콘텐츠 --> </main> <div id="footer-placeholder"></div> <!-- 루트 페이지 --> <script src="js/common.js"></script> <!-- 서브폴더 페이지 --> <script src="../js/common.js"></script> </body>

경로 자동 처리

페이지 위치 basePath 값 예시
루트 '' (빈 문자열) index.html
서브폴더 '../' company/overview.html

5. Wix 데이터 이관 작업

필수 작업 - 기존 Wix 사이트에서 운영 중인 데이터를 새 홈페이지로 이관해야 합니다.

이관 대상 데이터

구분 내용 비고
이미지 자산 제품 이미지, 로고, 배너, 아이콘 등 Wix Media에서 다운로드 필요
텍스트 콘텐츠 제품 설명, 회사 소개, 연혁 등 각 페이지별 복사
고객 사례 도입 사례, 고객사 로고, 후기 case-studies 페이지 반영
뉴스/공지 기존 게시된 뉴스 및 공지사항 news 페이지 반영
문서 파일 제품 브로슈어, 카탈로그 PDF 등 다운로드 링크 연결

이관 절차

1
Wix 미디어 라이브러리 백업 이미지, 동영상, 문서 파일 전체 다운로드
2
콘텐츠 정리 및 매핑 기존 페이지 ↔ 신규 페이지 콘텐츠 대응표 작성
3
이미지 최적화 및 적용 img 폴더에 정리, 각 페이지에 삽입
4
텍스트 콘텐츠 반영 각 HTML 페이지에 실제 내용 업데이트
5
검수 및 테스트 링크, 이미지, 반응형 확인

주의사항

  • Wix 이미지 URL은 외부 링크이므로 반드시 다운로드 후 로컬 저장
  • 이미지 파일명은 영문으로 통일 (한글 파일명 사용 X)
  • 고해상도 이미지는 웹 최적화 필요 (용량 1MB 이하 권장)

SEO (검색엔진 최적화) 이관 DEV

항목 현재 상태 필요 작업
메타 태그 Wix 자동 생성 각 페이지에 title, description, og 태그 직접 추가
URL 구조 Wix 형식 새 URL 구조로 변경됨 (SEO 영향 있음)
sitemap.xml Wix 자동 생성 새로 생성 필요
robots.txt Wix 자동 생성 새로 생성 필요
구조화 데이터 Wix 자동 생성 Schema.org JSON-LD 직접 추가 권장

백링크 및 리다이렉트 DEV

주의 - 기존 Wix URL로 연결된 외부 링크(백링크)가 있다면 검색 순위에 영향을 줄 수 있습니다.
상황 해결 방안
동일 도메인 유지 301 리다이렉트 설정 (서버 또는 호스팅 설정 필요)
새 도메인 사용 기존 도메인에서 새 도메인으로 301 리다이렉트
GitHub Pages 사용 커스텀 도메인 연결 가능 (CNAME 설정)

* GitHub Pages는 서버 리다이렉트를 지원하지 않음. 필요시 Netlify, Vercel 등 대안 검토

문의/신청 폼 처리 DEV

현재 상태

  • Wix Forms 사용 중
  • Wix 자체 백엔드로 데이터 저장
  • 이메일 알림 자동 발송

대안 필요

  • GitHub Pages는 정적 호스팅 (백엔드 없음)
  • 외부 폼 서비스 연동 필요
  • 또는 자체 백엔드 서버 구축

폼 서비스 대안

서비스 특징 비용
Formspree 간단한 HTML 폼, 이메일 전송 월 50건 무료
Google Forms 스프레드시트 연동, 무제한 무료
Netlify Forms Netlify 호스팅 시 자동 지원 월 100건 무료
자체 백엔드 완전한 커스터마이징 가능 서버 비용 발생
